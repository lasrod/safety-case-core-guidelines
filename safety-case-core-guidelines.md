## Core principles

1. Write claims that are specific and reviewable.
2. Separate claims, evidence, and assumptions.
3. Make uncertainty and limitations explicit.
4. Keep terminology consistent.

## Example guideline format

### SC.1 State the claim precisely

**Rationale**

A precise claim is easier to review, challenge, and support with evidence.

**Good example**

The system maintains safe braking performance for the defined operational design domain.

**Poor example**

The system is safe.
